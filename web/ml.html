Listes de diffusion

Informations diverses

Voici une liste des listes de diffusion en relation avec PostgreSQL. Pour vous abonner, ou vous désabonner, envoyez un mail à majordomo@postgresql.org avec le texte :

subscribe <nom_liste>

où <nom_liste> est un des noms de liste de diffusion ci-dessous. Vous pouvez envoyer plusieurs commandes d'abonnement en un seul message, chacune sur une ligne séparée.

Listes de diffusion actives

  • pgsql-admin
    Administration PostgreSQL
  • pgsql-advocacy
    PostgreSQL contre le reste du monde. Idées de promotion, etc.
  • pgsql-announce
    Liste d'annonces concernant PostgreSQL et différents logiciels de tiers
  • pgsql-bugs
    Si vous trouvez un bogue, merci de remplir le formulaire situé dans le code source de PostgreSQL (src/pgsql/doc/bug.template) et de l'envoyer par mail sur cette liste de diffusion.
    Si vous échouez lors de la compilation de PostgreSQL sur votre ordinateur, ce n'est pas la liste de diffusion dont vous avez besoin. Vous devez le rapporter sur pgsql-ports@postgresql.org.
    Si vous trouvez un bogue dans PostgreSQL et que vous l'avez déjà corrigé en créant un correctif pour PostgreSQL, merci de l'envoyer sur cette liste de diffusion ainsi que sur la liste pgsql-patches@postgresql.org.
  • pgsql-committers (Liste des développeurs)
    Les notifications des commits du CVS sont envoyées sur cette liste. Ne postez pas ici !!
  • pgsql-cygwin
    Pour tout problème concernant l'exécution, la compilation et l'installation de PostgreSQL sous Windows, en utilisant cygwin, voici l'endroit où chercher des informations.
  • pgsql-docs
    Discussions sur la documentation de PostgreSQL.
  •  pgsql-general
    Ceci est une liste de discussion générale pour les utilisateurs. En dehors des compilations, tests et bogues, la plupart des nouveaux utilisateurs ne veulent pas contribuer au développement ou à la documentation seront principalement intéressés par cette liste de diffusion.
    Toutes les questions ne concernant pas les bogues de la version de SQL doivent être restreint à cette liste.
    Toute question en relation avec l'installation mais qui n'incluent pas les bogues ou erreurs de compilation doivent être restreint à cette liste. Merci de noter que beaucoup de développeurs surveillent cette liste.
  • pgsql-hackers (Liste des développeurs)
    Les développeurs de PostgreSQL vivent ici.
    Cette liste permet de discuter des problèmes actuels de développement, des bogues maus aussi des nouvelles fonctionnalités.
    Si les personnes des autres listes ne connaissance pas la réponse à une question et qu'il est probable que seul un développeur pourra y répondre, vous pouvez y poser de nouveau votre question. Mais vous devez essayer ailleurs d'abord !
  • pgsql-interfaces
    Cette liste est utilisée pour discuter des interfaces PostgreSQL à l'exception de JDBC et d'ODBC.
  • pgsql-jdbc
    Cette liste est utilisée pour discuter de l'interface Java de PostgreSQL, JDBC.
  • pgsql-jobs
    Cette liste est utilisée pour envoyer des offres d'emplois en relation avec PostgreSQL.
  • pgsql-novice
    Aucune question n'est trop simple ici.
  • pgsql-odbc
    Cette liste est utilisée pour discuter de l'interface ODBC de PostgreSQL.
  • pgsql-patches (Liste des développeurs)
    Les nouvelles fonctionnalités et les correctifs doivent être envoyés ici.
  • pgsql-performance
    Cette liste est utilisée pour discuter des performances de PostgreSQL.
  • pgsql-php
    Utiliser PostgreSQL avec PHP.
  • pgsql-ports
    Porter et compiler PostgreSQL sur d'autres plateformes.
  • pgsql-sql
    Cette liste est un endroit pour discuter sur SQL.

Des informations sur les listes relatives au développement de PostgreSQL sont disponibles sur le site web des développeurs.
